The Raspberry Pi Pico and the RP2040 chip it runs on are a modern, low-cost microcontroller from the Raspberry Pi Foundation. At $4, the Pico competes directly with Arduino and ESP32 — but with better USB support, programmable I/O (PIO) for custom hardware protocols, and MicroPython as a first-class programming option. It is not a Linux computer (unlike Raspberry Pi 4) — it is a microcontroller for real-time hardware control.

## RP2040 vs ESP32 vs Arduino

| | RP2040 / Pico | ESP32 | Arduino Uno |
|---|---|---|---|
| Price | ~$4 | ~$4–10 | ~$25 |
| WiFi | No (Pico W: yes) | Built-in | No |
| Speed | 133MHz dual-core | 240MHz dual-core | 16MHz |
| USB | Native USB device | Via UART | Via FTDI |
| Language | MicroPython / C++ | Arduino C++ / MicroPython | Arduino C++ |
| PIO | Yes (custom protocols) | No | No |

The **Pico W** adds WiFi for ~$6 — making it an ESP32 alternative at similar price.

## Programmable I/O (PIO)

PIO is the RP2040's unique feature — dedicated state machines that run independently of the main CPU and can implement custom digital protocols in hardware. This makes WS2812 LED driving (addressable LEDs), custom serial protocols, and precise timing tasks far more reliable than software bit-banging on other microcontrollers.

## MicroPython

The RP2040 has first-class MicroPython support maintained by Raspberry Pi. For museum tech teams with Python skills (common when working alongside data/web teams), this is a lower barrier than C++.

## Use Cases

- Driving WS2812/NeoPixel addressable LEDs directly with the PIO engine
- USB HID device — appears as a keyboard/mouse/MIDI controller to a host computer
- Sensor aggregation node connected to a Raspberry Pi or PC via USB serial
- Low-cost custom MIDI controller or button interface
