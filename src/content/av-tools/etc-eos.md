ETC Eos is the dominant theatrical lighting console family — the standard in theatres, opera houses, and performing arts venues worldwide. If a museum has any kind of performance space, black box theatre, or theatrical lighting rig, it almost certainly uses an ETC Eos console (Ion Xe, Element, Ti, or Gio). Eos runs the theatrical cue-stack model: sequences of lighting states (cues) triggered by an operator pressing Go.

## Eos vs GrandMA

| | ETC Eos | GrandMA |
|---|---|---|
| Primary market | Theatrical / performing arts | Live events / touring |
| Cue model | Linear cue stack | Executor-based matrix |
| Fixture library | Excellent (ETC-maintained) | Excellent |
| Learning curve | Moderate | Steep |
| Price | Mid-range | Premium |
| Permanent install | Very common | Less common |

For a museum with a theatre or performance programme, Eos is more likely than GrandMA. Both output DMX, ArtNet, and sACN.

## Eos Family

- **Eos Ti:** Full-size flagship console
- **Ion Xe:** Mid-size, the most common in venues — 2 touchscreens, full-size fader wing
- **Element 2:** Entry-level, good for smaller venues
- **Nomad:** Software-only licence that runs on any Windows PC — useful for programming without a physical console

## Key Concepts

- **Cue list:** Ordered list of lighting states. Operator presses Go to advance
- **Palettes:** Named collections of values (colours, positions, beams) reused across cues
- **Groups:** Named sets of fixtures (e.g. "Front wash", "Cyc lights") for fast selection
- **Patch:** Mapping of fixture addresses to DMX channels/universes

## Use Cases

- Theatrical performances and events in a museum auditorium or performance space
- Ambient and architectural lighting with a theatrical cue stack
- Programming complex multi-fixture rigs for exhibitions that have a theatrical lighting component
- Nomad software for pre-programming before a console is available
