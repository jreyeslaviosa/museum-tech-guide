Processing is a flexible software sketchbook and programming language built for visual artists and designers. Created in 2001 at MIT Media Lab, it introduced the idea that code could be a medium for visual expression — and became the foundation for most creative coding education worldwide. If you've learned to code for visuals, you probably started here or in something inspired by it.

## Why It Still Matters

Processing's simplicity is its strength: `setup()` runs once, `draw()` loops every frame. That mental model — shared by p5.js, openFrameworks, and many others — is how creative coders think about time and interaction. Understanding Processing means understanding the grammar of creative coding.

## Use Cases

- Learning creative coding fundamentals before moving to more complex tools
- Rapid prototyping of generative visuals and interactive concepts
- Educational workshops for non-technical museum staff
- Simple interactive kiosk applications and data visualisations

## Processing vs p5.js

p5.js is Processing's JavaScript port — it runs in a browser, uses the same API, and is now arguably more popular for new projects. For permanent installations, Processing (Java) is more robust for long-running desktop apps; p5.js is better for browser-based or web-embedded pieces.
